| 2026-06-22 | v2.8.0 | Feature / Fix | **Admin project editor** — new **Projects** section in Admin Panel lists all submitted projects; each row expands to a full editor (name, short description, demo video link, git link, add/remove participants via search dropdown, check/uncheck tracks and sponsors, delete). Backend: `PUT /api/hackathons/:hid/projects/:id` (admin only). **Searchable user dropdown** — judges and project participants are now found via a debounced search-as-you-type input (queries `/api/admin/users?search=…`, max 20 results) instead of loading every user into a `<select>`. **AM/PM time display** — start/end times on the Overview now show as `9:00 AM – 5:00 PM`. **Clickable community links** — any URL in the Community & Support text renders as a live hyperlink. **Submission window** — broadened from strict same-day UTC to a 48-hour UTC window (event date + following day) to avoid timezone lock-out. | Update |
| 2026-06-19 | v2.7.0 | Feature | Hackathons now capture an event **date**, **start time**, **end time** and **location** (create form + admin edit). The **Overview** shows a **When & Where** card. **Projects can only be submitted on the hackathon's event date** — the submission API rejects off-day submissions (`403`) and the Submit page disables the form with a notice when it isn't the event day. Hackathons with no date set stay unrestricted (back-compat); new `event_date`/`start_time`/`end_time`/`location` columns auto-migrated. Tests → 73. | Update |

Columns auto-migrated. Tests → 71. | Update |
